Functor f => (a → b → c) → f a → f c

Takes a function and a functor, applies the function to each of the functor's values, and returns a functor of the same shape.

More info at Ramda map and addIndex.

mapIndexed
Since: v0.0.6
Parameters
fn (Function) The function to be called on every element of the input 'list'.
list (Array) The list to be iterated over.
Returns
Array: The new list.
Example
mapIndexed((val, idx) => idx + '-' + val, ['f', 'o', 'o', 'b', 'a', 'r'])
//=> ['0-f', '1-o', '2-o', '3-b', '4-a', '5-r']

mapP

src/mapP.js

Functor f => (a → Promise b) → f a → f Promise b

Takes a function and a functor, applies the function to each of the functor's values, and returns a functor of promises of the same shape.

Unlike Ramda map, only Array object can be implemented.

mapP
Since: v0.1.0
Parameters
fn (...Function) The function to be called on every element of the input list
list (Array) The list to be iterated over
Returns
Array: The list of promises
Example
//  Some promise returning functions
//  promised :: a → Promise b
const promised = (x) => Promise.resolve(x + 1);

mapP(promised, [1, 2, 3]) // => [2, 3, 4]
